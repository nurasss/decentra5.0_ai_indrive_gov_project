# Архитектура решения

## Цель

Найти потенциальное противоречие между нормой пользователя и нормами из корпуса НПА РК, затем дать объяснимый результат.

## Поток данных

### 1. Подготовка корпуса

- Источник: `adilet_parsed_texts.csv`
- Разбиение документов на чанки
- Сохранение чанков в локальную векторную базу `Chroma`

### 2. Retrieval

- Пользователь вводит норму
- Запрос преобразуется в embedding
- Из `Chroma` извлекаются наиболее похожие чанки

### 3. Базовый NLI-фильтр

- Модель: `facebook/bart-large-mnli`
- Назначение: быстро отсечь явные нерелевантные кандидаты
- Выход: бинарный флаг, стоит ли отправлять кандидата дальше

### 4. LLM Judge

- Модель: `llama3:8b` через `Ollama`
- Промпт: `few-shot chain-of-thought` + строгий JSON-ответ
- Выход:
  - `final_label`
  - `confidence_score`
  - `requires_human_review`
  - промежуточные reasoning steps

### 5. Маршрутизация

- `critical_conflict`: уверенное противоречие
- `human_review`: пограничный или рискованный случай
- `pass`: противоречие не подтверждено

### 6. Объяснение

- После judge вызывается генерация короткого объяснения
- Результат отображается в интерфейсе и может быть включён в отчёт

## Компоненты

### `index.py`

Строит локальную векторную базу из корпуса.

### `app.py`

Интерфейс `Streamlit` для демонстрации MVP.

### `nli_judge.py`

Реализует основной промпт-судью и JSON-парсинг ответа модели.

### `evaluation.py`

Считает метрики качества:

- `accuracy`
- `precision`
- `recall`
- `f1`
- `confusion matrix`
- сравнение `baseline` и `judge`
- проверку полного пайплайна

## Архитектурная схема

```text
Adilet CSV
   ->
Chunking
   ->
Embeddings
   ->
Chroma DB
   ->
Similarity Search
   ->
Baseline NLI
   ->
LLM Judge
   ->
Routing / Explanation
   ->
UI / Report
```

## Почему это MVP

- Есть рабочая логика поиска и классификации
- Есть API-подобный judge-модуль
- Есть интерфейс для демонстрации
- Есть метрики для оценки качества
- Есть путь масштабирования на полный корпус
